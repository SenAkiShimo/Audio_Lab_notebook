一、 声源定位（SSL）与到达时间差（TDOA）几何逆向推导

几何机理

声源定位（SSL，Sound Source Localization） 的核心初心是在不依赖摄像机视觉的前提下，纯粹通过声学方法计算出当前说话人的空间方位（通常表现为水平入射方位角 Azimuth \(\theta\)）。

在最通用的远场平面波（Far-field Plane Wave）假设下，声源距离阵列足够远。当声波斜向射入一个间距为 $d$ 的双麦克风元器件时，其到达两个通道的物理路径存在几何截断差。这导致两个接收通道的时域波形之间产生一个恒定的微秒级时间位移，即到达时间差（TDOA，Time Difference of Arrival），记为 $\tau$。

利用直角三角形的几何映射法则，时延值 $\tau$、物理间距 $d$、声速 $v$ 与入射偏角 $\theta$ 满足严格的非线性物理对齐关系：

$$ \tau = \frac{d \cdot \sin(\theta)}{v} $$

只要能利用算法在数字离散序列中精确计算出两路通道之间的客观时延 $\tau$，就能反向解调推导出发音人的真实方位角度：

$$ \theta = \arcsin\left(\frac{\tau \cdot v}{d}\right) $$

二、 传统时延估计：互相关函数（Cross-Correlation）的滑动积算机制

统计相关性

要在两个包含大量外界杂音的未知单声道序列 $x_1(t)$ 与 $x_2(t)$ 之间抓取出隐藏的相对时延量 $\tau$，最经典的统计学工具是互相关函数（Cross-Correlation）。其数学含义是将一个信号固定，另一个信号在时间轴上逐步平移，在每个移位点上执行全序列点乘积算：

$$ R_{x_1 x_2}(\tau) = \int_{-\infty}^{+\infty} x_1(t) \cdot x_2(t + \tau) \, dt $$

当移位量 $\tau$ 完美等于声波在空间中滑过的真实 TDOA 时，两个通道中的有用人声波形将处于波峰对波峰、波谷对波谷的绝对重合对齐状态，此时点乘积算得出的互相关能量值 $R_{x_1 x_2}(\tau)$ 将暴冲出全局最高绝对峰值（Peak Value）。

根据傅里叶变换的时移性质（时域的平移等于频域的复数相位旋转），时域复杂的平移卷积可以转化为频域简单的矩阵点乘。因此，工程上通常先将两路信号通过 FFT 转换为频域，求其交叉功率谱（Cross-Power Spectrum） $X_1(f) \cdot X_2^(f)$（其中 $$ 代表复数共轭），最后通过逆傅里叶变换（IFFT）回到时域寻找能量最高点：

$$ R_{x_1 x_2}(\tau) = \text{IFFT}\left( X_1(f) \cdot X_2^(f) \right) $$

三、 工业级行业天花板：GCC-PHAT 算子的白化去混响本质

硬核算法核心

普通的互相关算法在安静室内容易崩溃。因为真实房间里存在严重的室内混响（Reverberation），声音在墙壁上反复反射，会导致互相关矩阵在错误的时延位置上产生大量的虚警多径伪峰。同时，低频语音的自相关能量极高，会导致时域峰值曲线变得像小山丘一样极其宽大模糊，无法实现精度达到 1 度的精细分辨率。

为了攻克这个死穴，Knapp 和 Carter 提出了广义互相关-相位变换法（GCC-PHAT，Generalized Cross Correlation with Phase Transform）。它在频域的交叉功率谱中央强行挂载了一个加权白化函数（Weighting Filter）进行绝对滤除：

GCC-PHAT 核心白化算子公式：
在计算 IFFT 之前，将复数交叉功率谱除以其自身的绝对模长（振幅）： $$ R_{\text{GCC-PHAT}}(\tau) = \text{IFFT}\left( \frac{X_1(f) \cdot X_2^(f)}{|X_1(f) \cdot X_2^(f)|} \right) $$
去混响的硬核数学本质： 由于分子和分母在振幅上完全相等，相除之后，复数谱中所有频率通道的振幅能量全部被强行强平压制为 1。这种操作在信号处理中称为“白化滤波（Whitening）”。 经过白化处理后，交叉谱中彻底剥离了语音音色、宏观能量以及多径混响反射产生的振幅波动，仅仅死死保留了最为纯粹、干净的复数相位差信息（即纯粹的时间对齐时延）。 此时再通过 IFFT 返回时域，原本宽大模糊的能量曲线会坍缩为一个如钢针般极其锐利、干净的 δ 冲击函数尖峰，空间抗噪与去混响性能获得跨代级别的拉升。

四、 核心代码实现：手动推导 GCC-PHAT 算法并反向反调角度

代码演练

以下代码基于 NumPy 纯矩阵底层算子，完整还原演示了 GCC-PHAT 核心算法的全链路计算：接收两个包含微秒级空间时延的音频序列，进行 FFT 变换、频域共轭点乘、PHAT 幅度绝对归一化白化、IFFT 逆变换以及寻找脉冲峰值并解调角度 的全流水线工程仿真：

import numpy as np

def compute_gcc_phat_angle(signal_1, signal_2, mic_distance=0.02, fs=16000):
    """
    输入两路包含空间到达时延差(TDOA)的双通道离散时间序列，使用 GCC-PHAT 算法逆向计算声源入射角
    mic_distance: 麦克风间距 (米, d)
    fs: 数字系统采样率 (Hz)
    """
    v_sound = 340.0 # 物理声速 (m/s)
    N = len(signal_1)
    
    # 1. 跨越向频域转换：执行 1D 离散傅里叶变换 (FFT)
    X1 = np.fft.fft(signal_1)
    X2 = np.fft.fft(signal_2)
    
    # 2. 计算交叉功率谱：X1 点乘 X2 的复数共轭
    # 共轭矩阵保留了相互之间的相对相位偏移
    cross_power_spectrum = X1  np.conj(X2)
    
    # 3. 🧬 核心算子核心：执行 PHAT 相位变换绝对白化归一化
    # 振幅强行被除尽归一为 1，仅死死拦截并留存纯净的复数互相关相位信息
    # 添加 1e-9 防止低频无声区分母出现零溢出
    phat_weighting_filter = cross_power_spectrum / (np.abs(cross_power_spectrum) + 1e-9)
    
    # 4. 执行逆快速傅里叶变换 (IFFT)，回归到时域时间轴空间中
    # cc 矩阵代表每一个离散样本点移位距离上的互相关能量强度值
    cc = np.fft.ifft(phat_weighting_filter)
    cc_real = np.real(cc) # 提取实部
    
    # 5. 为了处理负时延（声音从左边传来），将 IFFT 输出执行半周期循环对位平移 (FFT Shift)
    # 移位后，数组中央索引对应时延 tau = 0
    cc_shifted = np.fft.fftshift(cc_real)
    
    # 生成与之精密对应的一维离散延迟样本点坐标轴 [-N/2, ..., +N/2-1]
    sample_delays = np.arange(-N//2, N//2)
    
    # 6. 寻找互相关能量图谱中的绝对全局最大值索引点 (Peak Search)
    peak_index = np.argmax(cc_shifted)
    estimated_sample_delay = sample_delays[peak_index] # 提取出产生最高共鸣的样本点延迟数
    
    # 7. 将离散的采样点单位还原映射回物理时间轴（秒 / 微秒）
    # 物理推导: tau = 样本延迟数 / 采样率
    estimated_tau = estimated_sample_delay / fs
    
    # 8. 根据前置声学物理几何公式，反向解调反三角函数，锁死空间偏置角 theta
    # sin(theta) = tau  v / d
    sin_theta = (estimated_tau  v_sound) / mic_distance
    
    # 边界物理合理性裁剪保护（防止噪声引发的数学超出 [-1, 1] 导致 arcsin 报错）
    sin_theta = np.clip(sin_theta, -1.0, 1.0)
    estimated_angle_deg = np.arcsigns = np.degrees(np.arcsin(sin_theta))
    
    print(f"[GCC-PHAT SUCCESS] 脉冲能量最高尖峰对齐点捕获成功值!")
    print(f" -> 时域针状尖峰检测位于延迟步: {estimated_sample_delay} samples | 物理换算 TDOA: {estimated_tau1e6:.2f} μs")
