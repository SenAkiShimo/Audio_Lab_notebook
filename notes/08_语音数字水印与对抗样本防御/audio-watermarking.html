一、 数字语音水印的安全性初心与工业级约束

合规与防伪

随着零样本变声（Voice Cloning）与自回归语音大模型（如 ChatTTS、CosyVoice）的全面爆发，声纹伪造与恶意音频诈骗已成为全球性安全威胁。数字语音水印（Audio Watermarking） 是一种在生成的音频序列中强行烙印特定隐含信息（如模型版权 ID、生成时间戳、用户序列号）的主动防御溯源技术。

一套工业级语音水印系统必须死死遵守三大核心悖论约束：

1. 绝对隐蔽性（Invisibility / Imperceptibility）： 水印信息的加入绝对不能破坏原始语音的质量。无论真人耳朵怎么复听，都无法察觉到任何背景沙哑或数码噪声，保持极高的平均意见得分（MOS）。

2. 极强的鲁棒性（Robustness）： 水印音频在经历网络分发、有损压缩（如强转低码率 MP3/Opus）、加性噪声污染、甚至扬声器播放后的“空气二次录音（Acoustic Path Recording）”等剧烈物理退化后，接收端依然能通过算法百分之百解码出加密信息。

3. 高信息容量（Payload Capacity）： 在每秒音频中能够稳定承载足够位数的二进制比特流数据，以确保能装得下复杂的防伪数字签名。

二、 时域空间隐写：最低有效位（LSB）算法机制

时域基础

最经典、最直观的时域数字隐写方案是 最低有效位（LSB，Least Significant Bit） 算法。

在标准的 16-bit PCM 数字语音中，每个采样点由 16 位二进制整数表示（数值区间为 -32768 到 32767）。这 16 位中，最左侧的高位比特（MSB）对声音振幅有着决定性的能量控制；而最右侧的第一位、第二位低位比特（LSB）代表的振幅扰动极小。

以 16-bit 的整型采样点为例，若其二进制序列为 1011 0101 1100 0010，如果将其最后一位（LSB）强行修改为 1 或 0，反映到声音物理世界中，仅仅引起了 $1 / 65536$ 的微弱振幅缩放。这种极微弱的波形微调完全淹没在系统自身的底噪中，人类听觉神经无法感知。算法正是通过将每个采样点的 LSB 依次替换为加密二进制序列的 0 或 1，实现信息的隐蔽写入。

缺陷：时域 LSB 算法缺乏抗压能力，一旦音频经历 MP3 有损压缩（高频被削），或者遭遇一点重采样，藏匿在最后一两位的二进制信息就会被破坏殆尽。因此，它通常只用于本地数字防篡改，不能用于复杂的网络溯源。

三、 终极隐蔽防护：心理声学频域绝对掩蔽效应（Masking Effect）

高级声学算法

为了兼顾隐蔽性与抗网络压缩的强鲁棒性，前沿算法转向了频域掩蔽。人类听觉系统在处理声波时，存在强大的人耳听觉掩蔽效应（Auditory Masking Effect）：

频域掩蔽机制（Spectral Masking）：
当空间中存在一个能量高、频带窄的强信号（称为掩蔽音 Masker，如 1000Hz 说话人共振峰）时，在其附近的特定频率带内（临界频带 Critical Band），人类耳朵的绝对听觉阈值（Hearing Threshold）会被强行向上拉高拉宽。 在这个被临时拓宽的对数包络阴影区内（称为掩蔽门限 Masking Threshold），只要藏入一个能量低于该门限的小信号（称为被掩蔽音 Maskee），人耳将百分之百判定为死寂。

基于此，高级水印算法（如基于 Spread Spectrum 扩频技术）首先对音频执行短时傅里叶变换（STFT）算出时频能量分布，依据 MPEG-1 心理声学模型（Psychoacoustic Model 1）动态求解出每一帧在各个频点上的复数掩蔽门限值值。随后，将带有强纠错能力的伪随机噪声序列（代表数字签名）进行振幅缩放，使其能量死死贴在掩蔽门限下方 3dB 处注入频域，最后通过 ISTFT 还原。由于信息被注入到了高能量的语义频段频域中，即使经历 Opus/MP3 的有损压缩，这些信息也会随着主语义一起被压缩保留，具备惊人的抗破坏生存力。

四、 核心代码实现：时域 LSB 水印全链路嵌入与无损解调

代码演练

以下代码基于整型移位运算法则，完整演示了如何在一维语音时间序列中将一串 8 位二进制字符串拆解、对齐移位嵌入各个采样点的 LSB 低阶位（Watermark Embedding），以及在接收端纯靠移位掩码反向剥离解调出原始隐藏信息（Extraction）的完整全链路工程仿真：

import numpy as np

def embed_lsb_watermark(pcm_signal_int16, secret_string_ascii):
    """
    [水印嵌入算子] 将一段字符串转换为二进制流，强制写入 16位PCM 信号的最后一维 LSB 比特位中
    """
    # 1. 将人类字符串转化为一维二进制比特流 (如 'A' -> [0,1,0,0,0,0,0,1])
    binary_secret = ''.join([f"{ord(char):08b}" for char in secret_string_ascii])
    bit_length = len(binary_secret)
    
    if bit_length > len(pcm_signal_int16):
        raise ValueError("音频序列过短，无法承载该长度的水印内容！")
        
    print(f"\n[EMBED ENGINE] 待隐写签名内容: '{secret_string_ascii}' | 转换为离散二进制流长度: {bit_length} bits")
    
    # 深度拷贝一份原始信号进行改写
    watermarked_signal = np.copy(pcm_signal_int16)
    
    # 2. 级联移位改写：逐帧将二进制 bit 覆盖到采样点的最后一位
    for i in range(bit_length):
        bit_to_hide = int(binary_secret[i])
        current_sample = watermarked_signal[i]
        
        # 3. 硬件级移位对齐操作：
        # current_sample & ~1 强行将二进制最后一位清零（变成偶数）
        # 随后 | bit_to_hide 将我们要隐藏的 0 或 1 贴到最右端
        watermarked_signal[i] = (current_sample & ~1) | bit_to_hide
        
    print(f"[EMBED SUCCESS] 最低有效位 LSB 覆盖完成 | 已加密污染采样点数: {bit_length}")
    return watermarked_signal, bit_length

def extract_lsb_watermark(watermarked_signal, bit_length):
    """
    [提取检测算子] 接收有源音频信号，在对齐的位置上通过二进制掩码，强制还原提取出版权信息
    """
    extracted_bits = ""
    
    # 1. 遍历被加密覆盖的目标时间区间
    for i in range(bit_length):
        sample = watermarked_signal[i]
        
        # 2. 二进制掩码提取：sample & 1 可以直接把最右侧的最后一位（LSB）剥离出来
        extracted_bit = sample & 1
        extracted_bits += str(extracted_bit)
        
    # 3. 将还原出的一维 01 字符串按照 8 位一组重新拼装为 ASCII 字符
    decoded_string = ""
    for i in range(0, len(extracted_bits), 8):
        byte_segment = extracted_bits[i:i+8]
        if len(byte_segment) == 8:
            decoded_string += chr(int(byte_segment, 2))
            
    print(f"[EXTRACT SUCCESS] 检测器解码退出 | 剥离还原的隐藏明文信息: '{decoded_string}'")
    return decoded_string

# 启动语音防伪与版权追溯仿真流水线
# 模拟一段硬件采集满的 16位有符号单声道 PCM 裸音频数据（1000个采样点）
mock_raw_pcm16 = np.random.randint(-10000, 10000, size=(1000,), dtype=np.int16)

# 定义要落盘的主动防伪签名（假设为模型出厂编号）
secret_signature = "MODEL_7B_#902"

# 执行全闭环攻防对抗
watermarked_audio, payload_len = embed_lsb_watermark(mock_raw_pcm16, secret_signature)
extracted_signature = extract_lsb_watermark(watermarked_audio, payload_len)
