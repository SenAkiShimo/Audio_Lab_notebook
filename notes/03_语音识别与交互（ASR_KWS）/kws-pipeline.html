一、 什么是 KWS 及其在边缘端的终极约束

业务场景

KWS（Keyword Spotting，关键词检测 / 语音唤醒） 的任务是在连续不断的实时音频流中，精准检测出特定的敏感词（如“嗨，Siri”或“天猫精灵”）。KWS 是整套 AI 语音交互的“看门狗”，必须在设备未唤醒时在芯片底层常驻运行（Always-on）。

由于寄生在低功耗的微控制器（MCU）或 DSP 芯片上，KWS 具有极其苛刻的三大物理约束：

1. 极低的功耗（Power Drain）： 必须控制在毫瓦（mW）级别，绝对不能因持续检测而导致移动设备严重发热或电池迅速耗尽。

2. 极低的延迟（Latency）： 从用户说完唤醒词到系统做出点亮屏幕等响应，总延迟通常要求控制在 200ms 以内。

3. 极低误唤醒率（FAR）与高唤醒率（FRR）： 工业级标准通常要求在日常环境噪声下，24小时误唤醒次数小于 1 次，同时正常唤醒率保持在 95% 以上。

二、 流式音频处理与时间滑窗（Sliding Window）

时序机制

与大模型一次性吞吐整段音频不同，KWS 面对的是无始无终的环形音频双缓冲区（Ring Buffer）。它在时间轴上使用流式滑窗（Streaming Sliding Window）机制运行：

环形时序滑窗运转拓扑：
1. 时域滑窗： 硬件麦克风每采集满 100ms 的纯 PCM 裸数据，就通过中断将其压入缓冲区，同时踢出最早的 100ms 数据。
2. 特征平移： 系统通常维护一个固定为 1 秒至 1.5 秒的时频特征平移窗口（例如包含 100 帧 Log-Mel 频谱特征的矩阵）。每当新一帧特征算子生成，特征矩阵就整体向左平移，新特征覆盖最右侧。
3. 流式推理： 神经网络以极高的频次（如每隔 100ms 一次）对这个 1 秒长的时间窗口特征矩阵进行一次前向推理，实时输出当前窗口包含唤醒词的置信度概率 $P_{\text{keyword}}$。

三、 工业级后处理流控：置信度平滑与防误触

工程算法

深度学习模型在单帧上的输出概率通常存在剧烈的数值抖动。如果直接使用“单帧概率超过阈值（如 $P > 0.8$）就直接判定唤醒”，在工程上会引发毁灭性的误唤醒灾难（如咳嗽声、电视广告声引发的突发尖峰）。

因此，工业级 KWS 流水线必须挂载严谨的后处理流控模块（Post-processing Pipeline）：

1. 时间窗平滑（Time-window Smoothing）： 不看单帧最高值，而是计算连续 $N$ 帧（如最近 300ms 内）概率值的移动平均值（Moving Average）或积算滑窗能量，消除瞬时虚警噪声。

2. 平滑窗双阈值判决法（Dual-Threshold Match）： 设置上限阈值 $T_{\text{high}}$ 和下限阈值 $T_{\text{low}}$。只有当特征流概率持续越过 $T_{\text{low}}$ 达到一定时长，且期间最高值突破 $T_{\text{high}}$，同时伴随局部能量突变时，硬件控制器才会向主 CPU 投递中断信号，宣布唤醒成功。

四、 核心代码实现：模拟流式 KWS 检测与后处理

代码演练

以下代码使用 Python 显式还原了工业级 KWS 底层的流式数据滚入、模型置信度抖动、以及移动平均平滑滤波算法的完整全链路仿真：

import numpy as np

class KWSPostProcessor:
    def __init__(self, high_threshold=0.85, low_threshold=0.4, smooth_window_size=5):
        """
        初始化工业级 KWS 后处理流控器
        high_threshold: 触发唤醒的硬阈值极限
        low_threshold: 维持激活状态的软底线
        smooth_window_size: 移动平均滑动窗口大小
        """
        self.high_threshold = high_threshold
        self.low_threshold = low_threshold
        self.window_size = smooth_window_size
        self.prob_history = [] # 存储最近几帧的模型概率值
        
    def process_frame_prob(self, raw_model_prob):
        """
        流式接收当前帧模型的原始概率，通过滑动窗滤波后输出最终的唤醒判决
        """
        # 1. 维护固定大小的环形历史队列
        self.prob_history.append(raw_model_prob)
        if len(self.prob_history) > self.window_size:
            self.prob_history.pop(0)
            
        # 2. 计算滑动时间窗内的移动平均值，压制概率抖动尖峰
        smoothed_prob = np.mean(self.prob_history)
        
        # 3. 工业级双阈值流控状态机判决
        is_triggered = False
        if smoothed_prob >= self.high_threshold:
            # 均值突破上限，且队列内最近有真实高爆发
            if np.max(self.prob_history) >= self.high_threshold:
                is_triggered = True
                self.prob_history = [] # 触发唤醒后清空历史，防止二次重触发
                
        return is_triggered, smoothed_prob

# 🧬 模拟流式运行场景
processor = KWSPostProcessor()
# 模拟一段长音频流推理，前 10 帧是环境噪音（低概率），第 11 帧是噪声引发的瞬时突变高概率，14-18帧是真正的连续发音唤醒词
mock_stream_probs = [0.02, 0.05, 0.01, 0.03, 0.02, 0.04, 0.01, 0.02, 0.05, 0.02, 
                     0.92, # 第11帧：瞬时单帧突变噪声（若没有后处理，这里就会触发误唤醒！）
                     0.12, 0.05, 
                     0.86, 0.89, 0.91, 0.88, 0.85, # 第14-18帧：真实的连续发音唤醒词
                     0.22, 0.04]

for i, raw_prob in enumerate(mock_stream_probs):
    triggered, smooth_p = processor.process_frame_prob(raw_prob)
    print(f"时间步 t={i:02d} | 原始输出: {raw_prob:.2f} | 平滑后概率: {smooth_p:.2f} | 唤醒判定: {triggered}")
